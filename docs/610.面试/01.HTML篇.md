# 面试题目

> 以下为总结的常考的HTML面试题

## 1.src和href的区别

src属性用于指定需要嵌入到文档中的外部资源，而href属性用于指定需要跳转到的链接地址。

## 2.对HTML语义化的理解

HTML语义化是指在编写HTML代码时，使用恰当的标签和属性，以便更准确地描述页面内容和结构，从而提高页面的可读性、可维护性和可访问性。HTML语义化的主要目的是让页面更加易于理解和解释，不仅对人类用户有帮助，也对搜索引擎和其他机器程序有帮助。 
 
具体来说，HTML语义化的几个重要方面包括： 
1. 使用恰当的标签：不仅要使用正确的标签，还要根据页面内容和结构选择合适的标签，如使用h1-h6标签表示标题，使用p标签表示段落，使用ul和li标签表示列表等。 
2. 使用恰当的属性：在标签中使用正确的属性，如使用alt属性描述图片内容，使用title属性提供提示信息等。 
3. 建立正确的文档结构：使用正确的标签和属性，建立良好的文档结构，使页面更易于理解和解释。 
4. 提高可访问性：使用恰当的标签和属性，使页面更易于被屏幕阅读器等辅助技术解释和访问。 
 
总之，HTML语义化是一种良好的编码习惯，可以提高页面的可读性、可维护性和可访问性，使网站更加易于理解和解释。

## 3.DOCTYPE的作用

DOCTYPE是Document Type的缩写，用于告诉浏览器当前HTML文档使用的是哪个HTML版本的规范，浏览器在解析HTML文档时会根据DOCTYPE声明来确定使用哪种文档类型定义（DTD），从而正确地渲染页面。 
 在HTML5中，DOCTYPE声明的格式比较简单，只需要写`<!DOCTYPE html>`即可。这个声明告诉浏览器，当前文档使用的是HTML5规范，浏览器会根据HTML5规范来解析文档。

## 4.script标签中的defer和async的区别

defer和async都是用于异步加载JavaScript代码的属性，它们的主要区别如下： 
 
1. 加载顺序：当浏览器遇到带有defer属性的script标签时，会继续解析HTML文档，同时异步下载JavaScript文件，但是JavaScript文件的执行要等到HTML文档解析完成之后才会执行，即在HTML文档的DOMContentLoaded事件之前执行。而带有async属性的script标签则是异步下载JavaScript文件，下载完成后立即执行，不保证执行顺序。 
 
2. 执行顺序：当页面中有多个带有defer属性的script标签时，它们的执行顺序是按照它们在HTML文档中出现的顺序执行的，而带有async属性的script标签则是下载完成后立即执行，不保证执行顺序。 
 
3. 兼容性：defer属性在HTML4.01中就已经被定义，可以在所有浏览器中使用，但是在IE9及以下版本中有一些限制。而async属性是HTML5中新增的属性，只能在较新的浏览器中使用。 
 
::: note 

defer属性适用于需要保证JavaScript代码按照HTML文档顺序执行的情况，而async属性适用于不需要保证执行顺序的情况，但需要尽快下载并执行JavaScript代码的情况。
:::

## 5.常见的meta标签有哪些

常见的meta标签有以下几种： 
1. viewport：用于设置移动设备的视口（viewport），包括宽度、缩放比例等。 
2. keywords：用于设置网页的关键字，有助于搜索引擎优化（SEO）。 
3. description：用于设置网页的描述，也有助于SEO。 
4. author：用于设置网页的作者。 
5. robots：用于设置搜索引擎爬虫的行为，如是否允许爬取、是否允许索引等。 
6. content-type：用于设置网页的字符编码。 
7. refresh：用于设置网页自动刷新的时间间隔。 
8. http-equiv：用于模拟HTTP头部信息，如设置网页的编码、缓存策略等。
9. apple-mobile-web-app-capable：用于设置Web应用程序在iOS设备上的显示方式。 
10. og:title、og:description、og:image等：用于设置网页在社交媒体上的展示信息，如Facebook、Twitter等。 
这些meta标签可以帮助网页提供更多的信息，从而提高用户体验和SEO效果。

## 6.HTML5有哪些更新

1. 新的语义化标签：HTML5新增了一系列的语义化标签，如`<header>`、`<nav>`、`<article>`、`<section>`、`<aside>`等，这些标签可以更清晰地描述页面结构和内容，有助于搜索引擎优化和无障碍访问。 
2. 表单控件：HTML5新增了一些表单控件，如日期选择器、颜色选择器、搜索框、范围选择器等，这些控件可以让用户更方便地输入数据。 
3. 视频和音频：HTML5新增了`<video>`和`<audio>`标签，可以直接在网页上播放视频和音频，不需要使用Flash或其他插件。 
4. Canvas绘图：HTML5新增了`<canvas>`标签，可以使用JavaScript在网页上绘制图形、动画等。 
5. Web存储：HTML5新增了localStorage和sessionStorage两种Web存储方式，可以在浏览器本地存储数据，不需要使用Cookie。 
6. Web Workers：HTML5新增了Web Workers，可以让JavaScript代码在后台运行，不影响页面的响应性能。 
7. Web Socket：HTML5新增了Web Socket，可以在浏览器和服务器之间建立持久连接，实现实时通信。 
8. 地理位置API：HTML5新增了Geolocation API，可以获取用户的地理位置信息，可以用于LBS等应用。

## 7.img的srcset属性

img的srcset属性用于指定图片的多个源文件，以便浏览器根据不同的设备屏幕大小和分辨率选择最合适的图片进行显示，从而提高页面加载速度和用户体验。srcset属性的语法格式如下：
``` HTML
<img src="default-image.jpg" 
     srcset="image-1x.jpg 1x, image-2x.jpg 2x, image-3x.jpg 3x"
     alt="image description">
```
其中，srcset属性的值由多个源文件和对应的像素密度比（x倍数）组成，每个源文件和像素密度比之间用空格分隔。浏览器会根据设备的像素密度选择最合适的源文件进行加载，比如在高清屏幕上会选择像素密度比更高的图片进行加载，以保证图片的清晰度和显示效果。 
 
除了像素密度比之外，srcset属性还可以使用w单位来指定图片的宽度，比如：
``` HTML
<img src="default-image.jpg" 
     srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
     sizes="(max-width: 600px) 400px, (max-width: 800px) 800px, 1200px"
     alt="image description">
```
其中，sizes属性用于指定图片在不同屏幕宽度下的显示大小，可以根据需要设置不同的媒体查询条件。这样，浏览器会根据设备屏幕宽度和图片宽度来选择最合适的图片进行加载。

## 9.说一下Web  Worker

Web Worker是HTML5中的一项技术，它允许在Web应用程序中创建后台线程，以便在主线程不被阻塞的情况下执行复杂的计算任务。Web Worker的主要特点包括： 
 
1. 在后台线程中执行任务，不会影响主线程的性能和响应速度。 
2. 可以通过消息传递机制与主线程进行通信，以便在任务完成后将结果返回给主线程。 
3. 可以创建多个Web Worker，以便同时执行多个任务。 
 
Web Worker的使用方法如下： 
 
1. 创建一个Worker对象，指定后台线程执行的JavaScript文件路径。 
2. 在后台线程中编写JavaScript代码，实现需要执行的任务。 
3. 在主线程中通过postMessage()方法向后台线程发送消息，以便传递任务参数。 
4. 在后台线程中通过onmessage事件监听主线程发送的消息，并根据消息内容执行任务。 
5. 在后台线程中通过postMessage()方法向主线程发送消息，以便返回任务结果。 
6. 在主线程中通过onmessage事件监听后台线程发送的消息，并根据消息内容处理任务结果。 
 
Web Worker只能在支持HTML5的浏览器中使用。同时，由于Web Worker在后台线程中执行任务，因此无法访问主线程中的DOM和全局变量，需要通过消息传递机制与主线程进行通信。

## 10.HTML5的离线存储怎么使用，它的工作原理是什么？

HTML5的离线存储是一种可以使Web应用程序在离线状态下正常工作的技术。它的使用方法和工作原理如下： 
 
使用方法： 
1. 在HTML文档中使用manifest属性指定一个描述文件，描述文件中列出需要离线储存的资源列表。 
2. 在描述文件中列出需要离线储存的资源，包括HTML文档、CSS文件、JavaScript文件、图片、音频、视频等。 
3. 用户第一次访问网页时，浏览器会下载描述文件中列出的资源，并将它们存储在本地缓存中。 
4. 当用户离线访问网页时，浏览器会从本地缓存中加载存储的资源，而不是从服务器重新下载。

 <a href="https://developer.mozilla.org/zh-CN/docs/Web/Manifest">关于manifest的详细使用说明</a>

工作原理： 
1. 用户访问网页时，浏览器会检查HTML文档中是否包含manifest属性，如果包含则下载描述文件。 
2. 浏览器根据描述文件中列出的资源列表下载对应的资源，并将它们存储在本地缓存中。 
3. 当用户离线访问网页时，浏览器会从本地缓存中加载存储的资源，而不是从服务器重新下载。 
4. 当用户在线访问网页时，浏览器会检查描述文件中列出的资源是否有更新，如果有更新则重新下载，否则继续使用本地缓存中的资源。 
 
需要注意的是，离线存储并不能完全替代服务器的资源管理和更新机制，因此在使用离线存储时，需要谨慎考虑资源的更新机制和缓存策略，以保证网页的正确性和稳定性。

## 11.浏览器是如何对HTML5的离线储存资源进行管理和加载？

浏览器对HTML5的离线储存资源进行管理和加载的过程如下： 
1. 在HTML文档中使用manifest属性指定一个描述文件，描述文件中列出需要离线储存的资源列表。 
2. 当用户第一次访问网页时，浏览器会下载描述文件中列出的资源，并将它们存储在本地缓存中。 
3. 当用户离线访问网页时，浏览器会从本地缓存中加载存储的资源，而不是从服务器重新下载。 
4. 当用户在线访问网页时，浏览器会检查描述文件中列出的资源是否有更新，如果有更新则重新下载，否则继续使用本地缓存中的资源。 
5. 当用户清除浏览器缓存时，浏览器会同时清除离线储存的资源。 
需要注意的是，浏览器对离线储存资源的管理和加载只是一种优化策略，不能完全替代服务器的资源管理和更新机制。因此，在使用HTML5离线储存时，需要谨慎考虑资源的更新机制和缓存策略，以保证网页的正确性和稳定性。

## 12.title与h1的区别，b与strong的区别，i与em的区别

- `<title>`标签用于定义网页的标题，是在浏览器标签栏中显示的文本，也是搜索引擎抓取网页信息的重要元素。 
- `<h1>`标签用于定义网页的主标题，是页面内容的重要组成部分，也是搜索引擎判断页面内容的重要指标。  
- 区别： `<title>`标签用于定义网页标题，`<h1>`标签用于定义网页主标题，它们的作用和位置不同。 
 
b与strong的区别： 
- `<b>`标签用于定义文本的粗体显示，是一种视觉效果的标记。 
- `<strong>`标签用于定义文本的强调显示，是一种语义效果的标记，表示文本的重要性或紧急性。 
- 区别： `<b>`标签只是为了视觉效果而强制文本加粗，而`<strong>`标签则是为了强调文本的重要性或紧急性，具有更强的语义意义。 
 
i与em的区别： 
- `<i>`标签用于定义文本的斜体显示，是一种视觉效果的标记。 
- `<em>`标签用于定义文本的强调显示，是一种语义效果的标记，表示文本的重要性或紧急性。 
- 区别： `<i>`标签只是为了视觉效果而强制文本斜体，而`<em>`标签则是为了强调文本的重要性或紧急性，具有更强的语义意义。

## 13.iframe有哪些优点和缺点

iframe是一种HTML标签，用于在网页中嵌入其他网页或文档。它有以下优点和缺点： 
 
优点： 
1. 可以将其他网页或文档嵌入到当前网页中，实现页面的动态更新和内容的实时展示。 
2. 可以实现页面的分块，将不同的内容分别放在不同的iframe中，提高页面的可维护性和可扩展性。 
3. 可以实现跨域通信，例如在一个网站中嵌入另一个网站的内容，或在一个域名下的子页面中嵌入另一个域名下的页面。 
 
缺点： 
1. iframe会增加网页的加载时间和带宽消耗，因为每个iframe都需要加载独立的HTML文档。 
2. iframe会影响网页的搜索引擎优化，因为搜索引擎通常不会将iframe中的内容视为当前页面的内容。 
3. iframe中的内容无法被父页面的JavaScript直接访问，需要通过特殊的技术来实现跨域通信。 
4. iframe中的内容可能会被其他网站滥用，例如通过iframe中的表单提交恶意数据。 
 
总之，iframe具有一些优点和缺点，需要根据实际需求和场景来选择使用。

## 14.label的作用是什么？如何使用？

`<label>`标签用于将表单元素与标签文本关联起来，使得用户可以通过点击标签文本来触发表单元素的操作。例如，当用户点击`<label>`标签的文本时，关联的`<input>`元素就会被选中或激活。这提高了表单的可用性和易用性。 
 使用`<label>`标签很简单，只需要将`<label>`标签的for属性设置为关联表单元素的id属性即可。例如，以下代码将一个文本框和一个标签关联起来：
``` HTML
<label for="username">Username:</label>
<input type="text" id="username" name="username">
```
在上面的代码中，`<label>`标签的for属性设置为"username"，与`<input>`元素的id属性相同，这样当用户点击"label"标签的文本时，就会激活关联的`<input>`元素

## 15.Canvas和SVG的区别

Canvas和SVG都是用于在网页上绘制图形的技术，但它们有几个重要的区别： 
 
1. Canvas是基于位图的渲染技术，而SVG是基于矢量图形的渲染技术。Canvas通过JavaScript在画布上绘制像素，而SVG则使用XML描述图形。 
 
2. Canvas适用于动态图像，例如游戏、动画和数据可视化等。SVG适用于静态图像，例如图标、图表和地图等。 
 
3. Canvas绘制的图像无法缩放而不失真，因为它是基于像素的。SVG绘制的图像可以缩放而不失真，因为它是基于矢量图形的。 
 
4. Canvas需要JavaScript来实现交互，例如点击、滚动和拖拽等。SVG可以使用CSS和JavaScript来实现交互。 
 
总之，Canvas和SVG都有各自的优缺点。Canvas适用于需要高性能和动态图像的场景，而SVG适用于需要可缩放和交互性的静态图像。

## 16.head标签有什么作用，其中什么标签必不可少？

head标签用于定义HTML文档的头部区域，包含了一些元数据和其他信息，如页面标题、样式表和脚本等。其中最重要的标签是`<title>`标签，它必不可少，用于定义页面的标题，同时也是在搜索引擎中显示的主要信息。其他常见的标签包括`<meta>`标签，用于定义网页的元数据，如网页的描述、关键字、字符编码等；`<link>`标签，用于引入外部资源，如样式表和图标等；`<script>`标签，用于定义JavaScript代码。总之，head标签包含了一些重要的元素，可以提供关于网页的重要信息，帮助浏览器正确地呈现页面

## 17.文档声明（DOCTYPE）有何作用？严格模式与混杂模式如何区分？他们有声明意义？

文档类型声明（DOCTYPE）指定网页中使用的HTML或XML版本，并告知浏览器如何呈现页面。它的作用非常重要，因为它有助于浏览器正确呈现页面并避免任何兼容性问题。 
 
严格模式和混杂模式是两种不同的页面渲染模式。严格模式是指浏览器按照W3C标准呈现页面，而混杂模式是指浏览器按照旧版浏览器的呈现方式呈现页面。可以通过DOCTYPE声明来区分这两种模式。如果DOCTYPE声明指定为严格模式，浏览器将按照W3C标准呈现页面，否则将按照混杂模式呈现页

## 18.浏览器乱码的原因是什么？如何解决？
浏览器乱码的原因可能有很多，其中一些常见的原因包括： 
 
1. 字符编码不匹配：如果网站使用的字符编码与浏览器使用的字符编码不同，就可能会出现乱码。 
 
2. 字体缺失：如果网站使用的字体在浏览器中不存在，就可能会出现乱码。 
 
3. 特殊字符处理不当：如果网站中包含特殊字符，但没有正确地处理这些字符，就可能会出现乱码。 
 
要解决浏览器乱码问题，可以尝试以下几种方法： 
 
1. 修改浏览器字符编码：在浏览器设置中修改字符编码，以确保与网站使用的字符编码匹配。 
 
2. 安装缺失的字体：如果浏览器中缺失了网站使用的字体，可以尝试安装相应的字体。 
 
3. 检查特殊字符处理：如果网站中包含特殊字符，可以检查是否已经正确地处理这些字符。 


## 19.渐进增强和优雅降级之间的区别？

渐进增强和优雅降级是两种不同的前端设计方法。渐进增强是指首先构建一个基本的网站，然后在此基础上逐步添加更高级的功能，以确保在低端设备和旧版浏览器上也能够正常使用。而优雅降级则是指首先构建一个高级的网站，然后在此基础上逐步降低功能，以确保在低端设备和旧版浏览器上也能够正常使用。总之，渐进增强强调的是从基础出发逐步增强，而优雅降级则强调的是从高级出发逐步降级。

## 10.说一下HTML5 drag API

HTML5 Drag API 是 HTML5 中新增的一组 API，用于支持拖放操作。通过 Drag API，可以实现将一个元素拖拽到另一个元素上的操作，同时还可以实现拖放文件等操作。 
 
Drag API 主要包括以下几个部分： 
 
1. draggable 属性：可以在 HTML 元素上设置 draggable 属性，将该元素设为可拖拽的。 
 
2. dragstart 事件：当用户开始拖拽一个元素时，会触发 dragstart 事件，可以在该事件中设置拖拽时所传递的数据。 
 
3. dragenter 事件：当一个可拖拽的元素进入到一个可放置的区域时，会触发 dragenter 事件。 
 
4. dragover 事件：当一个可拖拽的元素在一个可放置的区域上移动时，会触发 dragover 事件。 
 
5. dragleave 事件：当一个可拖拽的元素离开一个可放置的区域时，会触发 dragleave 事件。 
 
6. drop 事件：当一个可拖拽的元素被放置到一个可放置的区域时，会触发 drop 事件。 
 
7. dragend 事件：当一个可拖拽的元素被放置到一个可放置的区域后，会触发 dragend 事件。 
 
通过使用 Drag API，可以实现一些比较实用的功能，比如拖拽排序、拖拽上传文件等。
